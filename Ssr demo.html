CSR VS SSR

CLIENT-SIDE vs SERVER-SIDE RENDERING
IMPLEMENTED & COMPARED IN A PWA CONTEXT

SSR ⬛ SERVER-SIDE

El HTML se genera en el servidor antes de llegar al browser. El cliente recibe una página ya completa —lista para mostrar— sin necesidad de ejecutar JS para ver el contenido inicial. Usado por: Next.js, Nuxt.js, Astro, PHP, Django...

  • FCP (First Contentful Paint) instantáneo
  • Excelente SEO — crawlers ven contenido real
  • Funciona aunque JS esté deshabilitado
  • Mejor para contenido estático o poco dinámico
  • Cada navegación puede requerir request al server
  • Mayor carga en el servidor bajo tráfico alto
  • Time To Interactive (TTI) puede ser más lento
  • Requiere servidor Node/Python/PHP corriendo

CSR ◆ CLIENT-SIDE

JavaScript genera el HTML en el browser después de cargar. El servidor solo envía un HTML "vacío" con el bundle JS. El cliente hace fetch() y construye la UI. Usado por: React SPA, Vue SPA, Angular...

  • Interactividad rica sin recargar página
  • Actualizaciones en tiempo real (polling/websocket)
  • Servidor solo sirve archivos estáticos
  • Ideal para apps tipo dashboard o juego
  • FCP lento — el browser descarga JS antes de renderizar
  • SEO limitado sin SSR híbrido
  • Sin JS = página en blanco
  • Bundle JS puede crecer si no se optimiza

▸ CÓMO PROBAR OFFLINE + LIGHTHOUSE
1
INSTALAR CON LIVE SERVER
El SW necesita HTTPS o localhost. Usa VS Code Live Server, npx serve . o similar. Abre en Chrome → http://localhost:PORT
2
VERIFICAR SERVICE WORKER
DevTools → Application → Service Workers. Deberías ver sw.js como "Activated and running". Revisa Cache Storage para ver los assets cacheados.
3
PROBAR MODO OFFLINE
DevTools → Network → selecciona Offline en el dropdown. Recarga la página — debería cargarse desde caché. Si no hay caché aún, verás offline.html.
4
CORRER GOOGLE LIGHTHOUSE
DevTools → Lighthouse → selecciona categorías (Performance, PWA, SEO, Accessibility) → Analyze page load. Espera ~30s para el reporte. Importante: usa modo incógnito para resultados limpios.
5
INSTALAR COMO PWA
Chrome mostrará el ícono de instalar en la barra de URL. Click → Install → se abre como app nativa en una ventana separada, sin UI del browser. En iOS: Safari → Compartir → Añadir a pantalla de inicio.
